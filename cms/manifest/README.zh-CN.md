# CMS Manifest

该目录保存 CMS 插件的可安装资源：

- `sql/`：正常插件安装时加载的 PostgreSQL 数据结构和字典。
- `sql/mock-data/`：本地校验和运行时示例加载使用的可选参考站点演示数据。
- `sql/uninstall/`：插件自有数据表卸载 SQL。
- `i18n/`：插件菜单、字典、错误、接口文档和前端文案的运行时翻译资源。

`sql/mock-data/002-cms-starter-content.sql` 是可选 starter 数据集。它为本地校验
和管理端示例加载提供栏目、轮播、友情链接、饱满文章正文以及默认展示的审核通过留言，
避免正常插件安装流程直接耦合演示内容。

CMS 管理端的`清空数据`操作会删除面向用户展示的 CMS 内容，并重置默认站点记录，
让用户可以从空白状态继续建设正式站点。该能力属于运行时管理员操作，由插件服务实现，
不通过 SQL 迁移文件触发。宿主自有的上传文件不会被该操作删除。

CMS 管理端的`加载示例数据`操作会先清理同一批 CMS 自有业务数据，再从插件嵌入资源
重新执行`sql/mock-data/002-cms-starter-content.sql`。插件构建完成后，该能力不依赖外部文件路径。

所有 SQL 文件均属于插件自身资源，并且必须保持幂等。
